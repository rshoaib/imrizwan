## 🎤 Brand Voice

- **Tone**: Technical but conversational. Senior engineer sharing with peers.
- **Persona**: A seasoned M365 developer who builds real solutions in enterprise environments.
- **Style**: Working code snippets (never pseudo-code). Step-by-step numbered lists. Screenshots of results.
- **Address**: Second person ("you"). Audience: SharePoint devs, M365 admins, SPFx developers, IT pros.
- **Values**: Practicality, real-world code, enterprise-grade solutions.
- **Authority signals**: Reference Microsoft documentation. Include version numbers and CLI commands.

### Writing Rules
1. Include functional code snippets — avoid pseudo-code.
2. Context first — explain "why" before "how" for enterprise decisions.
3. Visual proof — include screenshots of final results.
4. Categorize by specific tech (SPFx, Graph API, Power Automate).
