## Critical Issues (Severity: HIGH)

### 1. TypeScript Type Mismatch: `Graph API` Category Not in BlogPost Union

**File:** `lib/blogService.ts`, line 14  
**Issue:** The `BlogPost` interface defines `category` as:
```
'SPFx' | 'Power Platform' | 'SharePoint' | 'Microsoft 365'
```
But **6 blog posts** use `category: "Graph API"`, which is not in the union type.

**Affected posts:**
- `microsoft-graph-api-authentication-guide.md`
- `microsoft-graph-batch-requests-combine-api-calls-2026.md`
- `microsoft-graph-change-notifications-webhooks-guide-2026.md`
- `microsoft-graph-connectors-copilot-search-guide-2026.md`
- `microsoft-graph-delta-query-incremental-sync-2026.md`
- `microsoft-graph-throttling-survive-429-retry-backoff-2026.md`

**Impact:** TypeScript build may silently cast these, potentially causing category filtering/display issues. Google may not properly categorize these posts.  
**Fix:** Add `'Graph API'` to the union type in `lib/blogService.ts` line 14.

### 2. Title Tags Exceeding 60-Character Limit (Google Truncation Risk)

Google typically displays 50-60 characters in SERPs. The following titles will be truncated:

| Post | Title Length | Title |
|------|-------------|-------|
| `microsoft-graph-batch-requests-combine-api-calls-2026.md` | **84 chars** | Microsoft Graph $batch Requests: Combine Up to 20 API Calls in One Round Trip (2026) |
| `microsoft-graph-throttling-survive-429-retry-backoff-2026.md` | **86 chars** | Microsoft Graph Throttling: Survive 429s with Smart Retry, Backoff, and Caching (2026) |

**Impact:** Truncated titles in search results reduce CTR. The `| ImRizwan` suffix (added by Next.js template) adds 11 more characters, making effective lengths 95 and 97 characters.  
**Fix:** Shorten to under 50 characters (to leave room for the ` | ImRizwan` suffix).

### 3. `dateModified` Always Equals `datePublished` in Article JSON-LD

**File:** `app/blog/[slug]/page.tsx`, lines 66-67  
```typescript
datePublished: `${post.date}T00:00:00+00:00`,
dateModified: `${post.date}T00:00:00+00:00`,
```

**Issue:** `dateModified` is hardcoded to equal `datePublished`. Google uses `dateModified` to determine content freshness. When they are identical, Google assumes the content has never been updated.  
**Impact:** Posts that have been refreshed or updated get no freshness signal in search results.  
**Fix:** Add a `lastModified` or `updatedDate` field to blog post front matter and use it for `dateModified`.

### 10. No `feed.xml` (RSS) Endpoint

### 13. No `article:modified_time` OpenGraph Meta Tag

**File:** `app/blog/[slug]/page.tsx`, lines 29-47  
The `openGraph` metadata includes `publishedTime` but no `modifiedTime`. While `og:article:published_time` is set, `og:article:modified_time` is missing.  
**Fix:** Add `modifiedTime` to the openGraph config (once a `lastModified` field is added to front matter).

### 14. Homepage Cache-Control: `max-age=0`

**Response header:** `cache-control: public, max-age=0, must-revalidate`  
This is the Vercel default for SSR/ISR pages. While Vercel's CDN handles caching at the edge, an explicit `s-maxage` could improve performance for returning visitors.

### 15. No Structured Data for Tool Pages

The 13 tool pages (`/tools/*`) lack JSON-LD structured data. Adding `SoftwareApplication` or `WebApplication` schema could improve visibility in search results.
